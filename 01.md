Para você vencer o 2º Hackathon Relâmpago (Seed Edition) do Serial Founders, preparei um planejamento estratégico detalhado com base em todas as regras, requisitos e dicas de ouro mencionadas nas fontes. 

Como a data limite para submissão é **hoje, sábado (30/05) até as 23:59**, vamos focar exatamente no que você precisa entregar.

### 1. Entenda o Conceito Central: A "SEED"
A regra de ouro deste hackathon é que **você não deve programar um aplicativo à mão**. O seu grande entregável é uma **SEED** – um arquivo de texto (geralmente `SEED.md`) contendo a "receita" detalhada do seu software. 
* O objetivo é que essa documentação seja "hidratada" (colada em um agente de Inteligência Artificial, como Claude ou Codex) e que a IA construa o projeto do zero sozinha. 
* **Atenção:** Entregar um projeto funcionando sem a SEED que o construiu resultará em **desclassificação imediata**.
* **Dica de Ouro:** Para estruturar muito bem a sua SEED, utilize como referência o `SPEC.md` do projeto *Symphony* da OpenAI ou o padrão RFC 2119 para documentar o que é obrigatório versus opcional.

### 2. O Desafio: Recriar o "Almanac"
Você deve escrever a SEED para recriar o app chamado **Almanac**. Ele é uma ferramenta de colaboração que funciona como os "comentários do Figma", onde um usuário faz o upload de um mockup HTML, compartilha o link com a equipe e eles podem deixar feedbacks ancorados (pins) em pontos exatos da tela.

### 3. As Restrições Técnicas Obrigatórias
A stack de tecnologias para a IA criar o código é livre, **exceto por três regras de infraestrutura** que serão usadas pelos juízes para testar a sua entrega:
1. O banco de dados **precisa** rodar no **Supabase**.
2. O site (deploy em produção) **precisa** estar hospedado na **Vercel**.
3. O sistema deve ter **Autenticação com o Google** para login.
*Se o seu projeto não funcionar com essas ferramentas, ele não passará na etapa de corte.*

### 4. Checklist de Funcionalidades (O Mínimo para não ser cortado)
Sua SEED deve instruir a IA a construir as seguintes funcionalidades obrigatórias (Jornada do Usuário):
* [ ] **Login:** Usuário consegue entrar com a conta Google.
* [ ] **Upload e Renderização:** O usuário consegue criar um projeto, fazer o upload de uma página HTML e vê-la renderizada na tela.
* [ ] **Histórico:** O usuário consegue ver uma lista dos projetos criados para acessá-los novamente.
* [ ] **Compartilhamento:** É possível gerar um link que permite a outros usuários verem a mesma página.
* [ ] **Feedback Ancorado:** Ao clicar em um ponto exato do HTML (mockup), cria-se um "pin" de comentário naquele local específico.
* [ ] **Visibilidade e Persistência:** Os "pins" ficam salvos (não somem ao recarregar a página) e qualquer colega com acesso ao link consegue ver a lista de feedbacks.

### 5. Como Ganhar o Hackathon (Os Diferenciais)
Cumprir o checklist acima garante que você passe do corte. Para ser escolhido o melhor e levar o prêmio principal de $125, instrua sua SEED a construir estes diferenciais:
* **Conversas e Threads:** Permitir que as pessoas respondam os comentários, reajam com emojis, editem, deletem e marquem o feedback como "resolvido" ou "reaberto".
* **Pins Avançados:** Funcionalidade de arrastar o pin para reposicioná-lo na tela, agrupar comentários que estejam muito próximos e fazer com que a âncora resista a mudanças de layout.
* **Histórico de Versões:** Sistema que permite criar diferentes versões (iterações/designs) da mesma página para que os membros possam comparar as opções.
* **Presença Ao Vivo:** Mostrar quem está visualizando a página no momento e adicionar um feed de atividades recentes.
* **Dashboard Organizado:** Ter contadores de atividade e status para os projetos (ex: ativo, arquivado, publicado).

### 6. Os 3 Entregáveis Finais
Quando a IA concluir o trabalho baseado na sua SEED, você precisará submeter exatamente estes três itens:
1. **Link do Repositório Git:** Um repositório acessível contendo o seu arquivo `SEED.md` (e outros `.md` de apoio, se houver).
2. **Link do Deploy em Produção:** A URL do projeto já hospedado na Vercel para ser testado pelos juízes.
3. **Vídeo de Demonstração:** Um link de um vídeo curto (pode ser uma demo simples) mostrando a você usando o app em funcionamento, provando que é possível fazer o login e usar todas as features exigidas.

Siga esta estrutura passo a passo, foque em criar a receita (`SEED.md`) com a máxima clareza possível e faça os testes garantindo a integração *Vercel + Supabase + Google*. Boa sorte na submissão!
